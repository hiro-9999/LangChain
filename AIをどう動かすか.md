大模型（LLM）アプリ開発は、従来のソフトウェア開発に「AIの特性」を組み合わせた新しい分野です。
3月24日の夜、明日の学習計画を立てるのにぴったりのトピックですね。

必要な知識と学習ステップを、エンジニアの視点で優先順位順に整理しました。

1. 基礎知識（土台）
まずは「AIをどう動かすか」の基本を押さえます。

Python / TypeScript: ほとんどのライブラリ（LangChain, LlamaIndexなど）がこの2言語で提供されています。特にPythonが主流です。

APIの利用方法: OpenAI (GPT-4) や Anthropic (Claude 3)、Google (Gemini) のAPIを呼び出し、パラメータ（temperature, top_p, max_tokensなど）を調整するスキル。

プロンプトエンジニアリング: AIから正確な回答を引き出すための「命令のコツ」（Few-shot, Chain-of-Thoughtなど）。

2. LLMオーケストレーション（応用）
AI単体ではできない「複雑な処理」をさせるための知識です。

LangChain / LlamaIndex: 先ほど紹介したフレームワークです。これを使えるようになると、開発スピードが劇的に上がります。

RAG (Retrieval-Augmented Generation): 自社データやPDFをAIに読み込ませる仕組み。これが現在の開発で最も需要が高い技術です（Deliver Results）。

ベクトルデータベース: テキストを数値化して保存・検索するデータベース（Pinecone, Milvus, Weaviate, または PostgreSQL + pgvector）。

3. エージェントとワークフロー
AIに「道具」を使わせる高度な技術です。

Function Calling / Tools: AIが自分で「今は計算機を使おう」「今は天気APIを叩こう」と判断して実行させる仕組み。

マルチエージェント: 複数のAI（エンジニア役、デザイナー役など）を協力させて一つのタスクを完了させる設計。

4. 評価と運用 (LLMOps)
作ったアプリを「製品」としてリリースするために不可欠です。

評価 (Evaluation): AIの回答が正しいか、ハルシネーション（嘘）がないかを自動・手動でチェックする手法。

コスト・遅延管理: API料金をどう抑えるか、回答までの待ち時間をどう短くするか。

💡 修正のポイント（日本語での学習目標の立て方）
もし上司やチームに「これを学びたい」と伝えるなら、こんな風に言うと説得力が増します（Earn Trust）。
