## Claude Code + Obsidian 個人知識庫
https://www.youtube.com/watch?v=dlnqJIgsiAg

Andrej Karpathy 提出了一個解法：不需要 RAG，不需要 vector database，就只是一個資料夾，讓 Claude Code 幫你自動整理成可查詢的 AI 知識庫。

在這支影片裡，你會學到：
• Karpathy LLM Wiki 的完整架構：raw / wiki / index / log 四層設計
• LLM Wiki Prompt 怎麼運作——它不是一個 prompt，是一份給 AI 的工作說明書
• Ingest 是什麼意思，以及它如何讓知識庫每次都長大一張網
• 五個你可以馬上開始建的知識庫（YouTube 逐字稿、會議記錄、AI 工具研究等）
• LLM Wiki vs RAG：什麼情況下用哪個
• 用 CLAUDE.md 把知識庫接進任何 Claude Code 專案
• 為什麼一個主題要建一個獨立 vault，而不是全部混在一起

如果你正在：
• 用 Claude Code 處理日常工作或內容創作
• 想讓 AI 真正「記住」你的知識和脈絡
• 或想把學習筆記、研究資料變成可以查詢的系統
