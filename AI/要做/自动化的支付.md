x402 是一种由 Coinbase 发起并开源的支付协议，旨在为 AI Agent（人工智能代理）提供一种原生、自动化的支付方式。它的核心思想是激活了长期被闲置的 HTTP 状态码 402 Payment Required，让 AI 在浏览网页或调用 API 时，能像人类扫码支付一样，自动完成小额支付。1. 什么是 x402？在传统的互联网协议中，HTTP 402 是一个“保留给未来使用”的错误代码。x402 协议赋予了它生命：机器支付标准： 它不是一个代币，而是一套通信规范。链上结算： 主要基于以太坊 Layer 2 网络（如 Base）或 Solana，使用 USDC 等稳定币进行结算。去中心化与零门槛： 支付过程中不需要注册复杂的 API 账户或绑定信用卡，AI 只要有钱包地址和余额，就能直接“买东西”。2. AI Agent 也能消费？（应用场景）过去，AI 只能处理免费信息。有了 x402，AI 拥有了“钱包”，可以自主完成以下操作：按需获取付费数据： AI 在搜索时发现一篇深度报告需要 $0.10$ 美元，它可以瞬间支付并读取，无需人类干预。购买计算资源： 一个 AI Agent 可以向另一个 AI 购买更强大的模型处理能力。自动化电商： 你的 AI 助理可以根据你的授权，直接为你在支持 x402 的平台上购买低价机票或预订服务。防止爬虫泛滥： 网站可以对 AI 爬虫设置极低额度的 x402 门槛（如每次请求 $0.001$ 美元），既不影响正常 AI 使用，又能通过成本过滤恶意攻击。3. 怎样使用？x402 的运作流程非常简洁，被称为“402 握手”：请求： AI Agent 请求一个受保护的资源（如一个付费 API）。挑战： 服务器返回 402 Payment Required 状态码，并附带支付详情（金额、接收地址、链信息）。支付： AI Agent 的钱包根据预设规则（如“单笔不超过 $0.5$ 美元则自动同意”）签署交易，发送 USDC。验证： AI 带着交易哈希（证明已付钱）重新发起请求。交付： 服务器验证交易后，返回 200 OK 并交付内容。总结x402 被视为 “AI 时代的经济基础”。它解决了 AI 无法拥有银行卡、无法点击支付弹窗的痛点。通过将支付变成一种简单的 HTTP 响应，它让 AI 真正从一个“聊天工具”转变为一个能独立参与市场交易的“数字个体”。

2. 集成 x402 的标准流程
一旦能访问文档，集成工作通常遵循以下逻辑：

第一步：环境配置
获取密钥：在管理后台申请 Client ID、Secret Key 以及用于签名验证的公钥/私钥。

设置回调地址 (Webhook)：在你的服务器上准备一个接收支付结果通知的接口，并将其注册到支付服务后台。

第二步：后端 API 对接
接入 x402 规范通常涉及以下核心接口：

创建交易 (Create Transaction)：调用 API 获取支付 URL 或 Token。

支付跳转：将用户引导至 DMM 的支付页面。

验证通知 (Verify Notification)：收到 Webhook 后，必须使用 SDK 或加密算法验证签名的真实性，防止掉包。

第三步：签名算法
支付服务对安全性要求极高，x402 可能会要求特定的签名方式。

注意：请务必在服务器端完成签名逻辑，切勿暴露 Secret Key 到前端。
