**Agentic AI（エージェンティック AI）とは、一言でいうと「自ら考え、判断し、行動する自律型AI」**のことです。

これまでのAI（チャット型など）との決定的な違いを、わかりやすく解説します。

1. 従来のAIとの違い
従来のAI（受動的）: 人間が「〇〇をやって」と指示し、AIが「答えを出す」だけで終わりでした。いわば**「優秀な辞書・電卓」**です。

Agentic AI（能動的）: 人間が「〇〇という目標を達成して」と伝えると、AIが自ら手順を考え、必要なツールを使い、完了まで実行します。いわば**「自律して働く部下・エージェント」**です。

2. 4つの大きな特徴
推論・計画（Reasoning & Planning）:
複雑な目標を小さなタスクに分解し、「まずAをやり、次にBをやる」という計画を自分で立てます。

ツールの利用（Tool Use）:
必要に応じて、自分でブラウザで検索したり、Pythonでコードを書いて実行したり、外部のAPI（Slackやメールなど）を叩いたりします。

自律性（Autonomy）:
一回一回人間に「次は何をしますか？」と聞かずに、目標達成までループを回して作業を続けます。

反省・修正（Reflection）:
途中でエラーが出たり、うまくいかなかったりした場合、自分で「なぜ失敗したか」を考え、やり方を修正します。

3. 具体的な活用例
エンジニアリング: 「このバグを直してテストを通しておいて」と指示すれば、コードを読み、原因を特定し、修正コードを書き、テストを実行してPRを出すまでを自動で行う。

市場調査: 「競合他社Aの新製品についてレポートをまとめて」と言えば、ウェブサイトを巡回し、情報を収集・比較し、グラフを作成して、PDFで保存するまでを完了させる。

4. なぜ今、注目されているのか？
LLM（大規模言語モデル）の推論能力が飛躍的に高まったため、単なる「文章生成」から「タスク遂行」へとAIの役割が変わってきたからです。


认知架构是驱动 AI Agent 行为、 决策和行动的核心架构，它聚焦于核心功能，由模型（Model）、
工具（Tool）和编排（Orchestration）组成，共同协作处理信息、做出决策并执行任务。AI Agent
通用架构及组件如图 1-1 所示。模型作为决策核心，编排使用 ReAct、CoT 等方法进行推理，工
具则提供与外部系统的连接。类比人类，模型可被视为大脑，工具可被视为双手，编排可被视为执
行机制。


「オーケストレーション（編排）」において ReAct や CoT を使うというのは、AIに「ただ答えさせる」のではなく、**「人間のように順序立てて考え、必要に応じて行動させる」**ためのテクニックのことです。Agentic AI（自律型AI）を実現するための核心的なエンジンとなる考え方です。それぞれわかりやすく解説します。1. CoT (Chain of Thought：思考の連鎖)**「結論を出す前に、一歩ずつ順序立てて考えさせる」**手法です。意味: 人間が数学の問題を解くときに書く「途中の計算式」のようなものです。やり方: プロンプトに「ステップバイステップで考えて（Let's think step by step）」と加えるのが最も有名です。メリット:複雑な計算や論理パズルの正解率が劇的に上がります。AIが「なぜその結論に至ったか」のプロセスが人間に見えるようになります。2. ReAct (Reason + Act：思考と行動の同期)**「考えて(Reason)、行動して(Act)、結果を観察する」**というサイクルを繰り返す手法です。意味: AIに「独り言（思考）」を言わせながら、「外部ツール（検索やDB）」を使わせるフレームワークです。流れ:Thought（思考）: 「次はGoogleで最新価格を調べる必要があるな」と考える。Action（行動）: 実際に検索APIを叩く。Observation（観察）: 検索結果の内容を確認する。（ループ）: 答えが出るまで 1〜3 を繰り返す。メリット:AIが自分の知識だけでなく、最新の外部情報を取り込めるようになります。現実世界のタスク（ファイルの操作、API送信など）を自律的にこなせるようになります。まとめ：CoT と ReAct の違い手法役割たとえ話CoT考えるだけ目を閉じて、頭の中だけで論理的に問題を解く。ReAct考えて動く参考書で調べたり、実験したりしながら問題を解く。



**RAG（Retrieval-Augmented Generation / 検索拡張生成）とは、一言でいうと「AIに、社内文書や最新ニュースなどの『外部知識の参考書』を読ませてから回答させる技術」**のことです。

AI（LLM）の最大の弱点である「知らないことでも自信満々に嘘をつく（ハルシネーション）」を防ぐための、現在最も実用的なソリューションです。

1. なぜRAGが必要なのか？
通常のAI（ChatGPTなど）は、学習した時点までの知識しか持っていません。そのため、以下の質問には答えられません。

「昨日発表された新製品の仕様は？」（最新情報の不足）

「わが社の社内規定では、経費精算はどうすればいい？」（非公開情報の不足）

これを解決するのがRAGです。

2. RAGの仕組み（3ステップ）
人間が試験で「教科書を見ながら回答する」プロセスと同じです。

Retrieval（検索）:
ユーザーの質問に関連する情報を、あらかじめ用意したデータベース（PDF、社内Wiki、DBなど）から探し出します。

Augmentation（拡張）:
見つかった情報を、ユーザーの質問と一緒にAIに渡します。
（例：「この資料を参考にして、質問に答えて」という指示を付け加える）

Generation（生成）:
AIがその資料を読んで、根拠に基づいた回答を作成します。

3. RAGのメリット
正確性が高い: 出典（ソース）を明示できるため、ユーザーが情報の正しさを確認できます。

低コスト: AIを一から再学習（ファインチューニング）させるには膨大な費用と時間がかかりますが、RAGならファイルをデータベースに入れるだけで済みます。

情報更新が簡単: データベース内の資料を差し替えるだけで、AIの知識を最新に保てます。

4. これまでの話題と繋げると？
関心を持たれている技術と組み合わせると、非常に強力なシステムになります。

FastAPI + RAG: 高速なAPIサーバーで、検索結果を即座にユーザーに返す。

Rust + RAG: 膨大なベクトルデータ（検索用データ）を高速に処理する。

Agentic AI + RAG: 自律型AI（ReAct）が、**「自分でRAGを使って資料を調べてから」**タスクを実行する。（これが現在の最先端です！）

💡 現場でよくある会話
もしチームで**「この案件、RAGでいける？」**という話が出たら、それは「モデル自体に覚えさせるんじゃなくて、外部のドキュメントを検索して答えさせる仕組みを作ればいいよね？」という意味になります。


AI Agent 的特点
在“A Survey on Large Language Model based Autonomous Agents”研究综述中，
AI Agent 的核心特点或能力主要体现在以下 3 个方面。
一是自主性（Autonomy）：无须持续的人工干预，可以独立完成复杂的任务流。
二是工具调用能力（Tool Invocation Capability）：通过整合外部资源，如 API、知识库等，
突破纯文本输出限制，实现与外界环境的有效互动。
三是动态适应性（Dynamic Adaptability）：利用短期记忆和长期记忆机制，根据反馈调整策
略，实现个性化服务。
对于具体类型的智能体核心特点详见 1.3.1 节内容，不再赘述。
